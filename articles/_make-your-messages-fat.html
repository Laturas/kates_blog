Make Your Messages Fat

February 17 2026

I am building a desktop mp3 player called Pinetree. While it's working and people are using it, as I'm building I'm having to deal with the frustration of a suboptimal design decision I made early on: I made my thread communication too fine-grained.

The Situation :tm:

For a decent mp3 player you need at least two threads because you need to handle the GUI and the playing of the audio. And of course, those threads have to communicate. The audio thread needs to know when the user does something on the GUI end, and the GUI needs information like which song is currently playing to accurately display it to the user.

This isn't a very difficult problem so I chose to implement a mailbox system. I built a vector that stores messages which can either be requests for data or a signal to update something (like pausing the current song) and then a mutex and condvar for the synchronization.

When all is said and done, you get an interface that looks something like this:

enum MessageToAudioThread { SetPaused(bool), SeekToPosition(usize), /* ... */ } fn audio_thread() { loop { /* TODO: Fill this out */ while !queue_is_empty() { } } }
